# Папки
Fundamental составляют базисные паттерны проектирования, лежащие в основе большинства подходов к проектированию программного обеспечения. Их задача — облегчить разработку устойчивых и удобочитаемых решений, позволяя эффективно управлять структурой и поведением системы.
- Delegation Pattern этот шаблон, при котором объект делегирует выполнение операции другому объекту, ответственному за нее.
- Functional Design подход к созданию программного обеспечения, ориентированный на обработку функций как первостепенных сущностей
- Immutable Interface этот паттерн предполагает использование неизменяемого API, где состояние объекта не изменяется после его инициализации.
- Interface паттерн, определяющий контракт для методов классов, гарантирующий унифицированное взаимодействие между классами независимо от их внутренней реализации.
- Marker Interface специальный вид интерфейса, используемый исключительно для маркировки классов или объектов определенными свойствами.
- Property Container представляет способ хранения набора атрибутов объекта отдельно от самого объекта
- Event Channel механизм передачи сообщений внутри системы посредством каналов событий

Creational паттернов проектирования предназначена для эффективного управления процессом создания объектов. Такие паттерны решают проблему, как создавать объекты правильно и гибко, особенно в ситуациях, когда важна инкапсуляция деталей реализации, уменьшение связности модулей и повышение уровня повторного использования кода.
- Abstract Factory определяет интерфейс для создания семейств связанных или взаимозависимых объектов без указания их конкретных классов.
- Builder отделяет построение сложного объекта от его представленного вида, что позволяет одному и тому же процессу строить разные представления.
- Factory Method предоставляет метод для создания объектов, который должен быть переопределён подклассами.
- Lazy Initialization позволяет отложить создание объекта до момента первого обращения к нему.
- Multiton обеспечивает существование ограниченного числа экземпляров класса.
- Object Pool создаёт пул готовых к использованию объектов, предотвращая избыточное выделение памяти и ускоряя работу программы за счёт многократного использования одних и тех же объектов.
- Prototype использует клонирование существующего объекта для создания нового экземпляра.
- Resource Acquisition Is Initialization (RAII) гарантирует автоматическое освобождение ресурсов.
- Singleton гарантирует наличие ровно одного экземпляра класса и предоставляет глобальную точку доступа к этому экземпляру.
