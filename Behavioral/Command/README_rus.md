# О шаблоне
Основная цель паттерна Command заключается в обеспечении гибкости и расширяемости системы путем отделения отправителя команды от её исполнителя.

# Основные принципы
- Интерфейс команды: Определяет общий интерфейс для всех классов-команд, включающих метод
- Конкретные команды: Реализуют конкретные действия, соответствующие интерфейсу команды.
- Получатель: Объект, выполняющий действие, вызванное командой.
- Клиент: Создает объект команды и передает его инвокеру.
- Инвокер: Запрашивает выполнение команды.

# Преимущества
- Отделение отправителя от получателя: Клиенты не зависят от конкретных реализаций команд.
- Поддержка отмены операций: Можно легко реализовать команду отмены (undo).
- Логирование и транзакционность: Легко вести историю выполненных команд и восстанавливать состояние системы.
- Параметризуемость поведения: Одна команда может использоваться многократно с разными параметрами.
- Упрощение добавления новых функций: Новые команды могут быть введены без изменения существующих классов.

# Применение в реальных проектах
- Графический интерфейс пользователя (GUI): Пользовательские действия (например, нажатия кнопок) преобразуются в команды, позволяя гибко управлять выполнением различных операций.
- Редакторы документов: Редактируемые программы часто используют паттерн Command для реализации истории изменений, поддержки отката и повторения действий.
- Игры: Игровые контроллеры и скрипты могут обрабатывать события путем отправки соответствующих команд.
- Удаленное управление устройствами:Устройства IoT управляются удаленно посредством посылки команд.

Таким образом, паттерн Command является мощным инструментом для структурирования приложений, особенно там, где важна поддержка отмены операций, расширяемость и гибкость архитектуры.
