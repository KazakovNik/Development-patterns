# О шаблоне
Паттерн Multiton является расширением известного паттерна проектирования Singleton («Одиночка»). Если Singleton гарантирует наличие ровно одного экземпляра класса, то Multiton позволяет иметь фиксированное количество экземпляров, каждый из которых ассоциирован с уникальным ключом.

# Основные принципы
- Класс поддерживает словарь (или ассоциативный массив), где каждому уникальному ключу соответствует экземпляр объекта.
- Каждый объект создается единожды и повторно используется при последующих обращениях.
- Экземпляры создаются лениво (lazy initialization): новый экземпляр создается только тогда, когда запрашивается впервые.
- Все операции с созданием объектов происходят внутри самого класса, контролируя доступ извне.

# Преимущества
- Возможность управлять несколькими объектами глобально и ограничивать число создаваемых экземпляров.
- Экономия ресурсов благодаря повторному использованию ранее созданных объектов.
- Простота управления множеством инстансов с помощью ключей.
- Поддержка уникальных состояний для каждого ключа.

# Применение в реальных проектах
Паттерн Multiton полезен там, где необходим контролируемый доступ к нескольким объектам с гарантированным количеством экземпляров. Примеры применения включают:
- Управление пулом соединений базы данных.
- Кэширование ресурсов с разными типами данных.
- Хранение конфигурации системы по различным профилям пользователей.
- Контроль над состояниями виджетов или компонентов интерфейса с различными режимами работы.

Таким образом, Multiton помогает разработчикам создавать управляемые и оптимизированные архитектуры приложений, минимизируя избыточность и обеспечивая контроль над ресурсами.
