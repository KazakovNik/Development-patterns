# О шаблоне
Маркерный интерфейс (Marker interface) — это шаблон проектирования, используемый в объектно-ориентированном программировании для обозначения класса, имеющего определённые характеристики или предназначенного для конкретного типа обработки, даже если сам класс не реализует никаких методов.

# Основные принципы
Паттерн реализуется следующим образом:
- Создается пустой интерфейс (без реализации каких-либо методов).
- Классы, которым необходимо обозначить наличие специфического поведения или свойств, реализуют этот интерфейс.
- Внешняя логика проверяет наличие интерфейса и обрабатывает объекты соответствующим образом.
Пример пустого интерфейса:
	public interface Serializable { }
Этот интерфейс ничего не определяет внутри себя, однако реализация класса такого интерфейса служит сигналом другим компонентам системы, что объект поддерживает сериализацию.

# Преимущества
- Простота: Не требует сложных реализаций.
- Четкость сигнализирования: Интерфейс чётко обозначает наличие определенного свойства объекта.
- Проверяемость: Легко проверить реализацию интерфейса с помощью операторов instanceof или аналогов.
- Безопасность типов: Использование интерфейсов позволяет создавать типобезопасные конструкции и избегать ошибок на этапе компиляции.

# Применение в реальных проектах
Примеры применения маркера-интерфейса:
- Классический пример — интерфейс Serializable. Реализация этого интерфейса означает, что объект можно преобразовать в поток байтов для передачи или хранения.
- Атрибуты вроде [Obsolete] позволяют помечать методы устаревшими, а также сами классы могут реализовать специальные интерфейсы для отображения определенных характеристик.
